# A) WINDOWS SETUP (Docker Desktop)

## Step 1: Install Docker Desktop

Download Docker Desktop:

[Docker Desktop Download](https://www.docker.com/products/docker-desktop/)

### Requirements

* Windows 10/11 (64-bit)
* WSL2 enabled
* Virtualization enabled in BIOS

---

## Step 2: Enable WSL2

Open **PowerShell as Administrator** and run:

```powershell
wsl --install
```

Restart system after installation.

Check WSL version:

```powershell
wsl -l -v
```

---

## Step 3: Verify Docker Installation

Open Command Prompt or PowerShell:

```powershell
docker --version
```

Expected Output:

```powershell
Docker version 26.x.x
```

---

## Step 4: Run Test Container

```powershell
docker run hello-world
```

Expected message:

```powershell
Hello from Docker!
```

---

## Step 5: Pull Ubuntu Image

```powershell
docker pull ubuntu
```

---

## Step 6: Run Ubuntu Container

```powershell
docker run -it ubuntu bash
```

Now you are inside the container.

Check Linux version:

```bash
cat /etc/os-release
```

Exit container:

```bash
exit
```

---

## Step 7: List Containers

```powershell
docker ps
```

Show all containers:

```powershell
docker ps -a
```

---

## Step 8: Start and Stop Container

Start container:

```powershell
docker start <container_id>
```

Stop container:

```powershell
docker stop <container_id>
```

## Ubuntu NVIDIA Docker Installation

### Step 1: Install NVIDIA Drivers

Check GPU:

```bash
nvidia-smi
```

---

### Step 2: Install NVIDIA Container Toolkit

```bash
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)

curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | \
sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
```

```bash
curl -s -L https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list | \
sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
```

```bash
sudo apt update
sudo apt install -y nvidia-container-toolkit
```

---

### Step 3: Restart Docker

```bash
sudo systemctl restart docker
```

---

### Step 4: Test NVIDIA Docker

```bash
sudo docker run --rm --gpus all nvidia/cuda:12.2.0-base-ubuntu22.04 nvidia-smi
```

---

## 1. What is Docker?

Docker is an open-source platform used to develop, deploy, and run applications inside lightweight containers.
It packages the application along with its dependencies, libraries, and configuration files.

### Key Features

* Fast deployment
* Portable environment
* Lightweight compared to Virtual Machines
* Easy scalability

---

## 2. Difference Between Virtual Machine and Container?

| Virtual Machine               | Container               |
| ----------------------------- | ----------------------- |
| Runs complete OS              | Shares host OS kernel   |
| Heavyweight                   | Lightweight             |
| Requires more RAM and storage | Requires less resources |
| Slow startup                  | Fast startup            |
| Uses Hypervisor               | Uses Docker Engine      |
| More isolated                 | Less isolated than VM   |

### Example

* VM → VMware, VirtualBox
* Container → Docker

---

## 3. What is a Docker Image?

A Docker Image is a read-only template used to create containers.

It contains:

* Application code
* Libraries
* Dependencies
* Environment variables
* Configuration files

### Example

```bash id="m95j91"
docker pull ubuntu
```

Here, `ubuntu` is a Docker image.

---

## 4. What is a Docker Container?

A Docker Container is a running instance of a Docker image.

It is:

* Lightweight
* Portable
* Isolated

### Example

```bash id="m6vt5i"
docker run -it ubuntu bash
```

This command creates and runs a container from the Ubuntu image.

---

## 5. What is Dockerfile?

A Dockerfile is a text file containing instructions to build a Docker image automatically.

### Example Dockerfile

```dockerfile id="7x75w8"
FROM ubuntu:latest

RUN apt update

CMD [\"bash\"]
```

### Common Instructions

| Instruction | Purpose               |
| ----------- | --------------------- |
| FROM        | Base image            |
| RUN         | Execute commands      |
| COPY        | Copy files            |
| CMD         | Default command       |
| WORKDIR     | Set working directory |

---

## 6. Difference Between Docker Hub and Docker Registry?

| Docker Hub                  | Docker Registry             |
| --------------------------- | --------------------------- |
| Public cloud repository     | Storage system for images   |
| Hosted by Docker            | Can be private or public    |
| Used to share images online | Used to store/manage images |
| Example: hub.docker.com     | Private enterprise registry |

### Example

```bash id="es0ddg"
docker pull nginx
```

Image is downloaded from Docker Hub.

---

## 7. Advantages of Containerization?

### Advantages

1. Lightweight
2. Faster deployment
3. Portability across systems
4. Better resource utilization
5. Easy scaling
6. Consistent development environment
7. Simplified application deployment

---

## 8. What are Basic Docker Commands?

| Command              | Purpose                 |
| -------------------- | ----------------------- |
| `docker --version`   | Check Docker version    |
| `docker pull ubuntu` | Download image          |
| `docker images`      | List images             |
| `docker run ubuntu`  | Run container           |
| `docker ps`          | Show running containers |
| `docker ps -a`       | Show all containers     |
| `docker stop <id>`   | Stop container          |
| `docker rm <id>`     | Remove container        |
